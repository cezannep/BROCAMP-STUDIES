# 🚀 Collaborative Learning Repository

Welcome to our Collaborative Learning Repository!

This repository is maintained by a group of students who are learning software development together. The goal is to document daily learning, share knowledge, practice collaboration using Git and GitHub, and build a centralized knowledge base that grows over time.

---

# 🎯 Mission

Learn. Document. Share. Improve.

Instead of keeping notes on personal devices, contributors add their learnings to this repository so everyone can benefit from them.

By contributing regularly, we aim to:

* Build a strong technical knowledge base
* Improve technical writing skills
* Practice Git and GitHub workflows
* Learn from each other's experiences
* Create a resource for future learners

# 🤝 Contribution Workflow

### 1. Pull Latest Changes

```bash
git pull origin main
```

### 2. Create a Branch

```bash
git checkout -b docs/topic-name
```

Examples:

```bash
git checkout -b docs/repository-pattern
git checkout -b docs/docker-basics
git checkout -b docs/react-hooks
```

### 3. Add Your Notes

Create or update the relevant files inside the repository.

### 4. Commit Your Changes

```bash
git commit -m "docs(nestjs): add repository pattern notes"
```

### 5. Push Your Branch

```bash
git push origin docs/topic-name
```

### 6. Create a Pull Request

Open a Pull Request and request review from the team.

### 7. Merge After Review

Merge only after review and approval.

---

# 📌 Commit Message Convention

Use meaningful commit messages.

### Documentation

```bash
docs(topic): add notes on repository pattern
```

### Feature

```bash
feat(project): add authentication module
```

### Fix

```bash
fix(notes): correct docker example
```

### Refactor

```bash
refactor(api): simplify service structure
```

### Chore

```bash
chore(repo): update folder structure
```

---

# 🚫 Repository Rules

* Do not push directly to `main`.
* Always create a branch before making changes.
* Keep notes simple and beginner-friendly.
* Avoid duplicate content.
* Verify information before adding it.
* Use meaningful file names.
* Respect review comments.
* Help others improve their contributions.
